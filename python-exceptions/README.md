# Python - Exceptions

This project is part of the curriculum at Holberton School. It focuses on handling errors and exceptions in Python, including safe printing, exception raising, and proper use of try/except/finally blocks.

## üìÅ Project Structure

Each file implements a specific task related to Python exception handling and error management.

### Files

- `0-safe_print_list.py`  
  Implements a function that safely prints x elements of a list using try/except without importing modules or using len().

- `1-safe_print_integer.py`  
  Implements a function that safely prints an integer using `"{:d}".format()` with exception handling, returning True if printed and False otherwise.

- `2-safe_print_list_integers.py`  
  Implements a function that prints the first x elements of a list but only integers, skipping other types silently, using try/except without importing or len().

- `3-safe_print_division.py`  
  Implements a function that divides two integers, printing the result inside a finally block, and handles division errors gracefully.

- `4-list_division.py`  
  Implements a function that divides element by element two lists, handling type errors, division by zero, and index errors with specific messages.

- `5-raise_exception.py`  
  Implements a function that raises a TypeError exception without importing any modules.

- `6-raise_exception_msg.py`  
  Implements a function that raises a NameError exception with a custom message.
